S/MIME Encryption

S/MIME provides message integrity, authentication, and non-repudiation (using X.509certificates) and protection against message interception. S/MIME employs public key encryption (PKI) for signing and encryption. Email which is encrypted and/or digitally signed by GoSecured Mail can be read in Outlook, Thunderbird and almost any other mail client, provided the user has the proper e-mail certificates installed. Certificates are automatically imported from incoming email and you can send (sign) emails with one button, and the GoSecured Mail system will automatically start the S/MIME Encryption with those recipients.

PDF Encryption

GoSecured Mail makes S/MIME very easy to use, but PDF encryption offers a light-weight alternative. When you just need to exchange secure email once, or a few emails over a longer period, or need to force the recipient to receive encrypted email, PDF encryption in GoSecured Mail is a perfect solution. You can have GoSecured Mail convert outgoing email—including all attachments—to a password encrypted PDF document using AES-128. Encrypted with a 116 bit or more password generated using a One Time Password (OTP) algorithm.

The password is then obtained by secure portal log-in. By using a separate channel for sending passwords, PDF encryption is almost as secure as full S/MIME encryption, and our OTP 116 bit or more passwords are strong enough to withstand a brute force attack. PDF encryption in GoSecured Mail is easy and intuitive. There is no need to specifically guide end-users.

Certificate Authorities (CA)

GoSecured Mail is its own certificate authority enabling complete protection, but is also compatible with any pre-existing CA server (like Microsoft CA or EJBCA). Compatibility extends to any pre-existing CA Server with certificates from external noncommercial Certificate Authorities and commercial (CAs) like Comodo, Verisign, and CACert. Alternatively, GoSecured Mail CA server allows GoSecured Mail to create certificates for internal and external users. Certificates can be transported to external recipients in a password secured format. Installation of the certificate in the recipient's mail client is direct. This allows you create your own private PKI with external recipients.
